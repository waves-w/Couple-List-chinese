# 键盘相关发热：可能原因与你在项目里的设置

## 1. 列表在键盘弹起时仍在刷新 + 每个 cell 做 AI 抠图（最可能）

**位置**：`HomeSwipeTableCell` → `loadAvatarForUser` → `ImageProcessor.shared.processAvatarWithAICutout`

**逻辑**：
- 键盘弹起时，Firebase / Core Data 仍可能推送更新。
- `NSFetchedResultsController` 收到变更后会刷新表格（插入/删除/更新行）。
- 每次 `cell.configure(with:)` 会调用 `loadUserAvatarForAssignIndex` → `loadAvatarForUser`。
- 头像未命中缓存时，会走 **AI 抠图**（`processAvatarWithAICutout`），非常耗 CPU。
- 结果：键盘开着时，后台仍在跑多次 AI 抠图 → 手机发烫。

**你设置/依赖的东西**：
- 首页列表用 `NSFetchedResultsController` 监听 Core Data。
- Cell 里用 `ImageProcessor.shared.processAvatarWithAICutout` 处理头像。
- 没有在「键盘可见」时限制或推迟这类重体力活。

---

## 2. 键盘上方的自定义 inputAccessoryView（中等可能）

**位置**：
- 引导页：`SetView`、`BootPageView`、`CheekBootPageView` 的 `setupKeyboardInputView`。
- 设置页：`UserAvatarViewController` 的 `setupKeyboardInputView`。

**逻辑**：
- 自定义 `keyboardInputView`（容器 + 输入框 + 之前还有 Done 按钮）作为 `inputAccessoryView`。
- 键盘每次出现/变化时，系统会布局、绘制这些 view。
- 若层级或约束偏重，会加重主线程负担，和第三方键盘一起更容易发热。

**你设置的东西**：
- `hiddenTextField.inputAccessoryView = keyboardInputView`。
- 每个键盘输入场景都有一套 60pt 高的白色条 + 圆角输入框。

---

## 3. 键盘通知的观察者（影响较小）

**位置**：
- `BootPageView`、`CheekBootPageView`：监听 `keyboardWillShow` / `keyboardWillHide`，只改 `isKeyboardShowing` 等布尔 + 一个 0.3s 延迟。

**逻辑**：回调里只做简单赋值和一次 `asyncAfter`，本身几乎不耗 CPU，一般不是发热主因。

---

## 4. 弹窗里的渐变 / 复杂视图（已部分优化）

**位置**：
- Add 弹窗：`backView` 是 `ViewGradientView`，之前 `textFiledView` 是 `BorderGradientView`（已改为纯色 UIView）。
- 其他弹窗里仍有 `BorderGradientView` / `ViewGradientView`。

**逻辑**：
- 键盘弹起会触发布局，若渐变层在 `layoutSubviews` 里频繁重算，会加重主线程。
- 你已在 `ViewGradientView` / `BorderGradientView` 里做了「仅 bounds 变化时更新」的优化；Add 弹窗的 Title/Notes 容器也改成了纯色，这块已经减轻。

---

## 5. 编辑时每键/每次变更的响应（当前很轻）

**位置**：
- `codeInputTextChanged`、`nameInputTextChanged` 等。

**逻辑**：目前实现几乎是空方法或极轻逻辑，不是发热来源。

---

## 总结：最值得先动的

| 可能原因                         | 你的相关设置                         | 建议 |
|----------------------------------|--------------------------------------|------|
| 键盘弹起时列表仍刷新 + cell AI 抠图 | FRC + `processAvatarWithAICutout`   | **键盘可见时跳过或推迟 AI 抠图**（见下） |
| 键盘上方自定义条                 | 多处 `inputAccessoryView`            | 保持简单，避免在 accessory 里做重布局/动画 |
| 键盘通知                         | 仅改布尔 + 延迟                      | 可保持现状 |
| 渐变视图                         | 已做 bounds 判断 + Add 弹窗已改纯色  | 已优化 |

建议优先做：**在「键盘可见」时，列表 cell 不执行 AI 抠图**（只用缓存或原图/默认图），等键盘收起再恢复，可显著减少键盘场景下的 CPU 和发热。
